# 寒假日程管理系统 - 多孩子版本使用说明

## 📚 功能说明

现在系统已经升级为**多孩子版本**！每个 HTML 文件都会对应一个独立的数据库，这样您可以为家里的每个孩子创建独立的日程管理系统。

## 🎯 核心特性

### 1. 独立数据存储
- **每个 HTML 文件 = 一个独立的数据库**
- 不同文件之间的数据完全隔离，互不影响
- 数据保存在浏览器的 localStorage 中，刷新页面不会丢失

### 2. 自动识别文件名
- 系统会自动读取 HTML 文件名
- 根据文件名生成唯一的存储键
- 在页面顶部显示当前管理的孩子名称

## 📝 使用方法

### 方式一：为每个孩子创建独立文件

1. **复制原始文件**
   - 找到 `寒假日程管理.html` 文件
   - 复制多份，每个孩子一份

2. **重命名文件**（推荐格式）
   ```
   寒假日程管理-小明.html
   寒假日程管理-小红.html
   寒假日程管理-小刚.html
   ```
   
   或者使用下划线：
   ```
   寒假日程管理_小明.html
   寒假日程管理_小红.html
   寒假日程管理_小刚.html
   ```

3. **打开对应文件**
   - 双击打开 `寒假日程管理-小明.html`
   - 页面顶部会显示 "📚 小明"
   - 这就是小明专属的日程管理系统

4. **添加日程**
   - 每个文件的数据完全独立
   - 在小明的文件中添加的日程，不会出现在小红的文件中

### 方式二：使用默认文件

如果文件名是 `寒假日程管理.html`（不带后缀），页面会显示 "📚 默认"

## 🔍 识别规则

系统会自动提取文件名中的关键信息：

| 文件名 | 显示名称 |
|--------|----------|
| `寒假日程管理.html` | 默认 |
| `寒假日程管理-小明.html` | 小明 |
| `寒假日程管理_小红.html` | 小红 |
| `寒假日程管理-大宝.html` | 大宝 |
| `schedule-Tom.html` | Tom |

**规则**：系统会提取最后一个 `-` 或 `_` 后面的内容作为显示名称

## 💾 数据存储说明

### 存储位置
- 数据保存在浏览器的 localStorage 中
- 每个文件对应一个唯一的存储键

### 存储键格式
```
winter_tasks_2026_寒假日程管理-小明
winter_tasks_2026_寒假日程管理-小红
winter_tasks_2026_寒假日程管理-小刚
```

### 数据安全
- ✅ 刷新页面不会丢失数据
- ✅ 不同文件的数据完全隔离
- ⚠️ 清除浏览器缓存会删除所有数据
- ⚠️ 更换浏览器需要重新输入数据

## 🎨 使用示例

### 场景：三个孩子的家庭

**文件结构：**
```
寒假日程管理/
├── 寒假日程管理-大宝.html
├── 寒假日程管理-二宝.html
└── 寒假日程管理-三宝.html
```

**使用流程：**

1. **管理大宝的日程**
   - 打开 `寒假日程管理-大宝.html`
   - 看到页面显示 "📚 大宝"
   - 添加大宝的英语课、数学作业等

2. **管理二宝的日程**
   - 打开 `寒假日程管理-二宝.html`
   - 看到页面显示 "📚 二宝"
   - 添加二宝的钢琴课、绘画课等

3. **管理三宝的日程**
   - 打开 `寒假日程管理-三宝.html`
   - 看到页面显示 "📚 三宝"
   - 添加三宝的游泳课、围棋课等

## 📊 数据管理

### 清除数据
- 点击页面上的 "清除数据" 按钮
- 只会清除**当前文件**的数据
- 其他孩子的数据不受影响

### 备份数据
如果需要备份数据，可以：
1. 打开浏览器的开发者工具（F12）
2. 进入 Application → Local Storage
3. 找到对应的存储键
4. 复制数据内容保存

### 恢复数据
1. 打开浏览器的开发者工具（F12）
2. 进入 Application → Local Storage
3. 找到对应的存储键
4. 粘贴之前备份的数据

## ⚠️ 注意事项

1. **文件名不要包含特殊字符**
   - ✅ 推荐：`寒假日程管理-小明.html`
   - ❌ 避免：`寒假日程管理@小明#.html`

2. **保持文件名稳定**
   - 修改文件名会导致无法访问原来的数据
   - 如果必须改名，需要先备份数据

3. **浏览器兼容性**
   - 推荐使用 Chrome、Edge、Safari、Firefox 等现代浏览器
   - 确保浏览器允许使用 localStorage

4. **数据安全**
   - 定期备份重要数据
   - 不要随意清除浏览器缓存

## 🚀 快速开始

1. 复制 `寒假日程管理.html` 文件
2. 重命名为 `寒假日程管理-[孩子名字].html`
3. 双击打开文件
4. 开始管理日程！

## 💡 提示

- 可以同时打开多个文件，在不同的浏览器标签页中管理不同孩子的日程
- 每个文件都是完全独立的，可以根据每个孩子的特点设置不同的课程和时间
- 页面顶部的彩色徽章会显示当前正在管理哪个孩子的日程
